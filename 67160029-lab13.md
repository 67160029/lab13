# 📝 คำถามท้ายปฏิบัติการ - วัดความเข้าใจ

## ส่วนที่ 1: useEffect Basics

### 1.1 Dependency Array

**A) ไม่ใส่ dependency array**
useEffect จะทำงานทุกครั้งที่ component render

- ใช้จริง: แทบไม่ใช้ เพราะเสี่ยงเกิด infinite loop

**B) ใส่ empty array []**
useEffect จะทำงานเพียงครั้งเดียวตอน component mount

- ใช้จริง: ใช้สำหรับ fetch API หรือ initialize ค่า

**C) ใส่ตัวแปร [var1, var2]**
useEffect จะทำงานเมื่อค่าที่อยู่ใน array เปลี่ยน

- ใช้จริง: ใช้กับการ fetch ข้อมูลตามค่า เช่น userId หรือ search

---

### 1.2 Cleanup Function

Cleanup function ใช้สำหรับล้าง resource ที่สร้างไว้ เพื่อป้องกัน memory leak

ตัวอย่าง:

- ลบ timer → clearInterval / clearTimeout
- ลบ event listener → removeEventListener

---

### 1.3 Infinite Loop

คำตอบ: **B และ C**

- B: เมื่อ count เปลี่ยน → useEffect ทำงาน → setCount → render ใหม่ → loop
- C: ไม่มี dependency → ทำงานทุก render → loop
- A: ทำครั้งเดียว ไม่เกิด loop

## ส่วนที่ 2: API Fetching

### 2.1 Component fetch users

```jsx
import { useState, useEffect } from "react";

export default function Users() {
  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    async function fetchUsers() {
      try {
        const res = await fetch("https://jsonplaceholder.typicode.com/users");
        if (!res.ok) throw new Error("fetch failed");
        const data = await res.json();
        setUsers(data.slice(0, 10));
      } catch (err) {
        setError(err.message);
      } finally {
        setLoading(false);
      }
    }

    fetchUsers();
  }, []);

  if (loading) return <p>Loading...</p>;
  if (error) return <p>Error: {error}</p>;

  return (
    <div>
      {users.map((u) => (
        <p key={u.id}>{u.name}</p>
      ))}
    </div>
  );
}
```

---

### 2.2 Race Condition

Race condition คือการที่ request เก่ามาช้ากว่า แต่ไปทับข้อมูลใหม่

วิธีป้องกัน:

1. ใช้ AbortController เพื่อยกเลิก request เก่า
2. ใช้ตัวแปร isMounted ตรวจสอบก่อน setState

---

### 2.3 ปัญหา search ทุกครั้ง

ปัญหา:

- ยิง API ถี่เกินไป
- เปลือง network
- แอปช้า

วิธีแก้:

- ใช้ debounce (delay ก่อนยิง API)

## ส่วนที่ 3: Cleanup & Side Effects

### 3.1 Timer Component

```jsx
import { useState, useEffect } from "react";

export default function Timer() {
  const [time, setTime] = useState(0);
  const [run, setRun] = useState(false);

  useEffect(() => {
    if (!run) return;

    const timer = setInterval(() => {
      setTime((t) => t + 1);
    }, 1000);

    return () => clearInterval(timer);
  }, [run]);

  return (
    <div>
      <h1>{time}</h1>
      <button onClick={() => setRun(!run)}>{run ? "Stop" : "Start"}</button>
    </div>
  );
}
```

---

### 3.2 Cleanup จำเป็นไหม

จำเป็น เพราะ:

- ป้องกัน memory leak
- ป้องกัน event ซ้ำ
- ป้องกัน timer ทำงานค้าง

## ส่วนที่ 4: Advanced Patterns

### 4.1 AbortController

AbortController ใช้สำหรับยกเลิก fetch request ที่ยังไม่เสร็จ

ตัวอย่าง:

```jsx
useEffect(() => {
  const controller = new AbortController();

  async function fetchData() {
    const res = await fetch(url, { signal: controller.signal });
    const data = await res.json();
    setData(data);
  }

  fetchData();

  return () => controller.abort();
}, []);
```

---

### 4.2 Custom Hook: useFetch

```jsx
import { useState, useEffect } from "react";

export default function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const controller = new AbortController();

    async function fetchData() {
      try {
        setLoading(true);

        const res = await fetch(url, {
          signal: controller.signal,
        });

        if (!res.ok) throw new Error("error");

        const result = await res.json();
        setData(result);
        setError(null);
      } catch (err) {
        if (err.name !== "AbortError") {
          setError(err.message);
        }
      } finally {
        setLoading(false);
      }
    }

    fetchData();

    return () => controller.abort();
  }, [url]);

  return { data, loading, error };
}
```
